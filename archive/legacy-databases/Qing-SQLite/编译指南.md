# SQLite 编译指南

## Windows 平台

### 方法 1: Visual Studio (推荐)

1. **安装 Visual Studio**
   - 下载：https://visualstudio.microsoft.com/
   - 安装时选择 "C++ 桌面开发"

2. **打开 Developer Command Prompt**
   - 开始菜单 → Visual Studio → Developer Command Prompt

3. **编译**
```cmd
cd C:\path\to\Qing-SQLite
cl sqlite3.c shell.c /Fe:sqlite3.exe /O2 /MD
```

4. **测试**
```cmd
sqlite3.exe test.db
SQLite version 3.52.0
Enter ".help" for usage hints.
sqlite> CREATE TABLE test(id INT, name TEXT);
sqlite> INSERT INTO test VALUES(1, 'Hello');
sqlite> SELECT * FROM test;
1|Hello
sqlite> .quit
```

### 方法 2: MinGW

1. **安装 MinGW**
   - 下载：https://www.mingw-w64.org/
   - 或使用 Chocolatey: `choco install mingw`

2. **编译**
```cmd
gcc sqlite3.c shell.c -o sqlite3.exe -O2 -lpthread
```

3. **测试**
```cmd
sqlite3.exe test.db
```

## 自定义编译选项

### 启用扩展功能

```bash
# JSON 支持
gcc sqlite3.c shell.c -o sqlite3 -O2 -DSQLITE_ENABLE_JSON1

# 全文搜索 (FTS5)
gcc sqlite3.c shell.c -o sqlite3 -O2 -DSQLITE_ENABLE_FTS5

# R-Tree 索引
gcc sqlite3.c shell.c -o sqlite3 -O2 -DSQLITE_ENABLE_RTREE

# 预写日志 (WAL)
gcc sqlite3.c shell.c -o sqlite3 -O2 -DSQLITE_ENABLE_PREUPDATE_HOOK

# 全部启用
gcc sqlite3.c shell.c -o sqlite3 -O2 \
  -DSQLITE_ENABLE_JSON1 \
  -DSQLITE_ENABLE_FTS5 \
  -DSQLITE_ENABLE_RTREE \
  -DSQLITE_THREADSAFE=1 \
  -lpthread -ldl
```

### 精简体积

移除不需要的功能以减小二进制大小：

```bash
gcc sqlite3.c shell.c -o sqlite3 -O2 \
  -DSQLITE_OMIT_COMPLETE \
  -DSQLITE_OMIT_EXPLAIN \
  -DSQLITE_OMIT_TRIGGER \
  -DSQLITE_OMIT_TCLVFS \
  -DSQLITE_DEFAULT_MEMSTATUS=0 \
  -DSQLITE_DEFAULT_WAL_SYNCHRONOUS=1
```

## 静态库编译

### Linux/macOS

```bash
# 编译静态库
gcc -c sqlite3.c -O2 -DSQLITE_ENABLE_JSON1
ar rcs libsqlite3.a sqlite3.o

# 使用静态库
gcc main.c -L. -lsqlite3 -o myapp
```

### Windows (MSVC)

```cmd
:: 编译静态库
cl /c sqlite3.c /O2 /MD
lib sqlite3.obj /OUT:sqlite3.lib

:: 使用静态库
cl main.c sqlite3.lib /Fe:myapp.exe
```

## 动态库编译

### Linux

```bash
# 编译共享库
gcc -shared -fPIC sqlite3.c -o libsqlite3.so -O2 -DSQLITE_ENABLE_JSON1 -lpthread

# 使用共享库
gcc main.c -L. -lsqlite3 -Wl,-rpath,. -o myapp
```

### macOS

```bash
# 编译动态库
gcc -dynamiclib sqlite3.c -o libsqlite3.dylib -O2 -DSQLITE_ENABLE_JSON1

# 使用动态库
gcc main.c -L. -lsqlite3 -Wl,-rpath,. -o myapp
```

### Windows (DLL)

```cmd
:: 编译 DLL
cl sqlite3.c /LD /Fe:sqlite3.dll /O2

:: 会生成 sqlite3.dll 和 sqlite3.lib
```

## 常见问题

### Q: 编译时提示 "pthread not found"

**A**: Linux/macOS 需要链接 pthread 库
```bash
gcc sqlite3.c shell.c -o sqlite3 -O2 -lpthread -ldl
```

### Q: Windows 下编译后运行提示 "缺少 VCRUNTIME140.dll"

**A**: 安装 Visual C++ Redistributable
- 下载：https://aka.ms/vs/17/release/vc_redist.x64.exe

### Q: 如何启用调试模式

**A**: 添加 `-g` 和 `-DDEBUG` 标志
```bash
gcc sqlite3.c shell.c -o sqlite3 -g -DDEBUG -O0
```

### Q: 编译后文件太大怎么办

**A**: 使用精简选项或 strip 命令
```bash
# Linux/macOS
strip sqlite3

# 或使用精简编译选项
gcc sqlite3.c shell.c -o sqlite3 -Os \
  -DSQLITE_OMIT_COMPLETE \
  -DSQLITE_OMIT_EXPLAIN
```

## 性能优化

### 编译优化级别

```bash
# O2: 标准优化（推荐）
gcc sqlite3.c shell.c -o sqlite3 -O2

# O3: 激进优化
gcc sqlite3.c shell.c -o sqlite3 -O3

# Os: 优化体积
gcc sqlite3.c shell.c -o sqlite3 -Os

# O0: 无优化（调试用）
gcc sqlite3.c shell.c -o sqlite3 -O0 -g
```

### 针对特定 CPU 优化

```bash
# Intel/AMD x64
gcc sqlite3.c shell.c -o sqlite3 -O2 -march=native

# ARM (Raspberry Pi 等)
gcc sqlite3.c shell.c -o sqlite3 -O2 -march=armv8-a
```
